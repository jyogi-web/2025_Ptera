## ドキュメントの読み方

各ドキュメントには以下のセクションが含まれます:

1. **概要**: 機能の簡単な説明
2. **目的**: なぜこの機能が必要か
3. **機能要件**: 必須機能とオプション機能
4. **技術仕様**: データモデル、API、実装例
5. **UI設計**: 画面設計とワイヤーフレーム
6. **セキュリティ**: Security Rulesなど
7. **テスト項目**: テストすべき項目
8. **実装の優先順位**: どの順番で実装するか

## ドキュメントの更新

開発を進めながら、適宜ドキュメントを更新してください。

- 新しい要件が追加されたら、該当するドキュメントに追記
- 実装中に発見した課題や改善点を記録
- 完了した項目にはチェックマークを付ける
